## Скрипт (7 минут)

### Слайд 1 — Проблема (30 сек)
> "В Казахстане более 13 миллионов носителей казахского языка. Но крупные модели — GPT-4, Gemini — допускают ошибки в казахских ответах. Облачные сервисы создают вопросы приватности и недоступны без интернета. Я решил эту проблему."

### Слайд 2 — Решение (45 сек)
> "KazGPT — локальный AI-ассистент. Java Spring Boot бэкенд, Ollama runtime для inference, MLX framework для fine-tune. Всё работает на моём ноутбуке без интернета. Демонстрирую."

### Слайд 3 — Обучение (1 мин)
> "Дообучил Qwen2.5-1.5B на казахских данных через MLX LoRA. KazQAD от ISSAI Назарбаев Университета требует HF-авторизации — использовал fallback датасет (multidomain-kazakh-dataset). 200 итераций. Train loss с 2.95 до 1.28, val loss с 3.45 до 1.98 — модель сходится. Качественно v2 уступает базовому 7B Qwen — это ожидаемо: 1.5B параметров против 7B, мелкий и шумный датасет, всего 200 итераций. Главное — pipeline работает, и в roadmap полноценное обучение с курированным датасетом."

## Q&A — заготовленные ответы

**"Чем отличается от ChatGPT?"**
> Локально (приватность, бесплатно, без интернета), специализация на казахском (fine-tune), open-source.

**"Это же обёртка над готовой моделью?"**
> Inference да, использую Qwen как foundation. Но я дообучил её на KazQAD — это инженерная часть. Показываю график loss.

**"Почему 200 итераций, а не больше?"**
> Trade-off между качеством и временем в рамках 2-дневного скоупа. Full fine-tune (5000+) — в roadmap.

**"Метрики качества?"**
> Train loss с 3.4 до 2.3 за 200 итераций. Val loss следует за train (нет overfit). Качественная оценка — `docs/before_after.md`.

**"Что в коде твоего, а что готового?"**
> Готовое: Qwen base, Spring Boot, MLX framework, Ollama runtime. Моё: вся интеграция (бэк + фронт + стрим), data pipeline, fine-tune конфигурация, demo-cache страховка, презентация.

**"Что если модель выдаст неправду?"**
> Низкая temperature, жёсткий system prompt запрещает галлюцинации. На демо использую заранее проверенные вопросы. В production нужен RAG (retrieval-augmented generation) — это в roadmap.

## План Б

| Что сломалось | Что делать |
|---------------|------------|
| Ollama не отвечает | Включить `kazgpt.demo-mode: true`, перезапустить бэкенд |
| Backend упал | `mvn spring-boot:run` в новом терминале |
| v2 не отвечает | Переключить в UI на "base" |
| Ноутбук не включился | Открыть `demo/demo_video.mp4` |
| Спросили вопрос вне кэша/тренировки | "Хорошее замечание, эта тема выходит за рамки текущего скоупа" |
