- Base модель (7B параметров) даёт более полные и связные ответы
- V2 модель (1.5B + LoRA) короче и стилистически беднее — ожидаемый результат для mini fine-tune (200 итераций) на ограниченных данных
- Для production качества необходимо: 5000+ итераций, курированный QA-датасет (KazQAD требует HF auth), batch_size > 1
- Главное достижение v2: pipeline работает, val loss сошёлся (3.45 → 1.98), модель адаптирована
